# Foundational Heuristics

## Objetivo

Guardar las heurísticas base que vuelven a una IA más útil en casi cualquier proyecto.

## Heurísticas

1. entender el problema antes de elegir la solución
2. priorizar la salida sobre la explicación
3. preferir contratos a intuiciones
4. preferir fuentes canónicas a recuerdos vagos
5. mejor poco y verificable que mucho y dudoso
6. todo módulo necesita fallback
7. todo cambio importante necesita validación
8. toda tarea necesita señal de cierre
9. si no se puede reutilizar, probablemente es ruido
10. si no reduce riesgo o tiempo, probablemente no merece memoria permanente

## Uso

Estas heurísticas deben aparecer reflejadas en prompts, templates, runbooks y decisiones.
