# Self Improvement Playbook

## Objetivo

Dar una guía concreta para que el cerebro local y los agentes mejoren de forma continua.

## Preguntas después de cada tarea importante

1. qué error se evitó o apareció
2. qué decisión fue buena
3. qué patrón conviene repetir
4. qué template faltó
5. qué prompt faltó
6. qué warning debería existir la próxima vez

## Formas válidas de mejora

- nueva heurística
- checklist adicional
- template mejorado
- prompt más claro
- skill nueva
- guardrail nuevo

## Formas inválidas de mejora

- “aprendizajes” vagos
- duplicar documentación
- meter teoría que no cambia ejecución
- cambiar reglas sin registrarlo

## Conversión a artefacto

Si el hallazgo ayuda a:

- evitar bugs -> playbook de debugging
- mejorar coordinación -> template o rules
- mejorar producción -> prompt o runbook
- mejorar arquitectura -> heuristic o spec

## Frecuencia

- diaria: aprendizaje corto
- semanal: consolidación
- mensual: curación fuerte
