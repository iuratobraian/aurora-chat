# Portability Rules

## Regla 1

No copiar el sistema como adorno. Debe adaptarse al proyecto nuevo.

## Regla 2

El cerebro local no se mueve como verdad absoluta. Se mueve como base de criterio y patrones.

## Regla 3

Las decisiones específicas del proyecto viejo deben separarse del criterio reusable.

## Qué se hereda

- procesos
- guardrails
- prompts
- heurísticas
- contracts
- estructura operativa

## Qué se adapta

- producto
- rutas
- archivos críticos
- blockers
- integraciones
- roster real del proyecto
