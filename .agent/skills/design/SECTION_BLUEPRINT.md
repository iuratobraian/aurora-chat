# Section Blueprint

## Cuándo usarlo

Para crear cualquier bloque nuevo de UI: home sections, panels, widgets, pages parciales, pricing blocks o módulos de comunidad.

## Blueprint

### 1. Propósito

- qué valor entrega la sección
- qué acción principal habilita

### 2. Ubicación

- en qué pantalla vive
- si es above the fold o below the fold

### 3. Jerarquía

- título principal
- soporte
- CTA principal
- CTA secundaria

### 4. Layout

- una columna dominante o grid
- cards o lista o tabla
- elementos de contexto lateral

### 5. Estados

- loading
- empty
- error
- success si aplica

### 6. Responsive

- mobile stacking
- tablet layout
- desktop layout

### 7. Datos

- origen
- frecuencia de actualización
- fallback permitido

### 8. Riesgo

- qué puede romper la experiencia si se implementa mal
