# UX UI System

## Problema actual

La interfaz mezcla demasiados estímulos: glass genérico, módulos apilados, navegación amplia y poca jerarquía editorial.

## Dirección visual

- premium sobrio, no futurismo ruidoso
- contraste alto, lectura clara, componentes consistentes
- menos efectos simultáneos
- más estructura, más contenido escaneable

## Reglas de diseño

1. Cada pantalla debe tener una acción principal clara.
2. El hero no puede competir con 5 widgets al mismo tiempo.
3. Cards y badges deben comunicar prioridad, no solo decorar.
4. El feed debe ganar legibilidad antes que espectacularidad.
5. La navegación debe orientar, no exhibir todas las features.

## Sistema visual base

### Tipografía

- principal: mantener `Plus Jakarta Sans`
- técnica: mantener `JetBrains Mono`
- usar pesos y tamaños para jerarquía real, no solo uppercase

### Color

- una paleta principal seria y consistente
- color de acento reservado para CTA, estado o señal importante
- evitar múltiples degradados compitiendo dentro del mismo viewport

### Layout

- una columna dominante por pantalla
- sidebars solo cuando agregan contexto real
- espaciado más amplio y consistente

## Mejoras prioritarias

1. Rehacer landing/home con propuesta de valor clara.
2. Simplificar `ComunidadView` en bloques legibles.
3. Reducir densidad visual de navegación y barras flotantes.
4. Reescribir pricing con beneficios y comparación clara.
5. Ajustar mobile con foco en lectura, CTAs y navegación.
