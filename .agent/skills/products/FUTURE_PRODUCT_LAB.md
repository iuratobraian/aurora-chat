# Future Product Lab

## Objetivo

Reservar un espacio canónico para futuras creaciones que no pertenecen a la web actual.

## Regla principal

Todo lo que viva en este frente:

- no modifica la web actual por defecto
- se diseña como vertical separada
- debe nacer con spec, arquitectura, datos, monetizacion y plan de integracion

## Verticales futuras contempladas

- plataforma de trading y MT5 automation
- app de growth para redes sociales
- super app generativa
- apps mobile nativas
- juegos mobile
- herramientas SaaS
- media y video systems

## Regla tecnica

Ninguna idea futura se implementa dentro del producto actual sin:

1. validar encaje
2. definir ownership
3. aislar dependencias
4. evaluar riesgo de complejidad
5. definir integracion o separacion

## Resultado esperado

Ideas futuras con conocimiento profundo, pero sin contaminar la base actual.
