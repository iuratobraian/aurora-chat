# Cross Project Mastery Playbook

## Objetivo

Dar a los agentes una ruta profesional para abordar proyectos distintos sin perder nivel.

## Secuencia obligatoria

1. clasificar el proyecto
2. abrir spec correcta
3. definir arquitectura minima viable
4. declarar datos y source of truth
5. definir release path
6. definir crecimiento y monetizacion
7. ejecutar por modulos cerrables

## Clasificacion rapida de proyecto

### Tipo A: contenido y comunidad

- feed
- perfiles
- comentarios
- reputacion
- moderacion

### Tipo B: workflow y productividad

- paneles
- CRUD
- permisos
- automatizaciones
- historial

### Tipo C: realtime y multiplayer

- presencia
- sockets
- salas
- sincronizacion
- eventos

### Tipo D: fintech o high trust

- dinero
- identidad
- cumplimiento
- trazabilidad
- soporte

### Tipo E: juego mobile

- core loop
- progresion
- economia
- retention
- live ops

### Tipo F: IA/media

- prompting
- providers
- aprobacion humana
- evaluacion
- costos

## Regla de arquitectura

Elegir lo mas simple que soporte:

- estabilidad
- crecimiento
- observabilidad
- handoff entre agentes

## Regla de calidad

No se mide por cantidad de features.
Se mide por:

- claridad del flujo
- robustez de datos
- seguridad
- performance
- adopcion

## Artefactos minimos por proyecto

- spec
- blueprint tecnico
- data plan
- risk register
- release checklist
- growth loop

## Donde dejar cada cosa

- specs y skills: `.agent/skills/`
- templates: `.agent/templates/`
- runbooks y planes: `.agent/workspace/plans/`
- coordinacion: `.agent/workspace/coordination/`
