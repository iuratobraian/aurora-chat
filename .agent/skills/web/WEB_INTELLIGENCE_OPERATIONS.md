# Web Intelligence Operations

## Objetivo

Dar a los agentes una forma profesional de extraer, organizar y reutilizar informacion publica de sitios, productos y ecosistemas externos.

## Principio

No hacer scraping ciego.
Hacer inteligencia web responsable.

## Reglas no negociables

1. respetar `robots.txt` cuando aplique
2. respetar terminos de uso y limites del sitio
3. no saltar auth, paywalls ni barreras tecnicas
4. no recolectar datos sensibles o privados
5. guardar solo lo necesario para el objetivo
6. citar fuente, fecha y contexto
7. transformar hallazgos en conocimiento util, no en ruido

## Tipos de trabajo

### 1. Site Extraction

Para sitios publicos:

- estructura
- urls clave
- pricing
- flujos
- copy
- features
- IA visible
- señales de conversion

### 2. Product Teardown

Para apps y plataformas:

- onboarding
- navegacion
- monetizacion
- engagement
- trust
- growth loops
- observabilidad visible

### 3. Architecture Inference

Inferir, sin inventar, como esta construido un sistema usando:

- engineering blogs
- docs oficiales
- carreras/jobs tecnicos
- conferencias oficiales
- changelogs y docs publicas

## Flujo operativo

1. definir objetivo
2. clasificar tipo de fuente
3. abrir template correcto
4. extraer urls y secciones
5. resumir hallazgos
6. separar hechos de inferencias
7. convertir en playbook para el repo

## Salidas esperadas

- dossier de sitio
- teardown de app
- teardown de juego
- brief competitivo
- patrones transferibles
- anti-patrones

## Regla final

Un agente de inteligencia web no colecciona paginas.
Construye conocimiento accionable para producto, growth, seguridad y arquitectura.
