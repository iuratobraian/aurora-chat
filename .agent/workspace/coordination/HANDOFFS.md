# 🤝 HANDOFFS (Protocolo de Relevo Estricto)

El trabajo asíncrono requiere contexto perfecto. Si debes abandonar una reparación a medias o si terminaste el Backend y le pasas el ticket al Frontend, el contexto mudo no sirve. **DEBES DEJAR UN HANDOFF AQUÍ**.

### PLANTILLA OBLIGATORIA DE RELAVO (Copia y pega esto al final de este doc):

```markdown
### 🏷️ [ID-DE-TAREA] - [Nombre Resumido]
**Agente Saliente:** [Tu Nombre/Módulo]
**Fecha/Hora:** [Timestamp]

1. **¿Qué estabas haciendo?**
   > (Ej: "Tratando de conectar Convex con el componente CardProfile")
2. **¿Cuál es el bloqueo actual? / ¿Por qué paras?**
   > (Ej: "Me saltó un error de TypeError en la linea 45. El Frontend espera un array pero el backend manda un object.")
3. **ZONA BLINDADA (No Tocar):**
   > (Ej: "La mutación de Convex ya fue auditada por seguridad. EL PROBLEMA ESTÁ EN EL FRONTEND. No re-diseñes el backend.")
4. **Próximo paso que harías si siguieras:**
   > (Ej: "Cambiaría la interfaz `UserProfile` en `lib/types.ts` para que coincida con el backend.")
```
