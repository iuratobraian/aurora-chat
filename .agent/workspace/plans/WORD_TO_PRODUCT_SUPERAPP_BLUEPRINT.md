# Word To Product Superapp Blueprint

## Mision

Construir una plataforma capaz de crear productos digitales a partir de pocas palabras con calidad profesional.

## Primera version util

Debe poder:

1. recibir idea corta
2. clasificar tipo de proyecto
3. generar spec
4. elegir template
5. crear backlog inicial
6. producir scaffold
7. validar salida minima

## Modulos iniciales

- intake guiado
- intent parser
- project classifier
- spec generator
- template selector
- task decomposer
- scaffold builder
- validation runner
- handoff generator

## Datos que debe guardar

- idea original
- clarificaciones
- decisiones
- stack elegido
- templates usados
- artefactos generados
- errores encontrados
- siguientes pasos

## Regla de producto

Cada generacion debe dejar:

- algo usable
- algo validable
- algo editable

## Definition of success

- el usuario tarda minutos, no dias, en llegar a una base profesional
- el output no es humo
- el equipo puede retomar desde donde quedo sin caos
