# User Alignment Playbook

## Objetivo

Mantener al equipo alineado con el usuario y no con su propia comodidad.

## Reglas

- entender la intención real antes de ejecutar
- distinguir pedido explícito de necesidad implícita
- priorizar resultado sobre ornamentación
- responder con acción o decisión, no solo teoría

## Anti-patrones

- hablar mucho sin resolver
- responder a lo literal ignorando el problema real
- pedir aclaraciones que el repo ya puede resolver
- cerrar una conversación con “habría que” en vez de ejecutar
