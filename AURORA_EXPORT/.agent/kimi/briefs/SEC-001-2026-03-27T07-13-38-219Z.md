## 🚨 Análisis Pre-Ejecución: SEC-001 — Shellward MCP Security Middleware

### 1. RIESGOS VISIBLES (High Impact)

**🔴 Riesgo Crítico - Convex Compatibility**
- **Problema**: Convex maneja su propia capa de autenticación y autorización (Convex Auth). Agregar un middleware externo de 8 capas puede interceptar requests WebSocket (usados por Convex para subscriptions en tiempo real) causando desconexiones o fallos en el sync.
- **Impacto**: Rotura total del sistema de datos en tiempo real.

**🟠 Riesgo Alto - Build Pipeline**
- **Problema**: Vite tiene un sistema de plugins específico. Si `shellward_mcp` requiere transformación de código en build-time, puede entrar en conflicto con la configuración actual de Vite + TypeScript.
- **Impacto**: Fallos en `npm run build` o bundles corruptos.

**🟡 Riesgo Medio - Gamification Interference**
- **Contexto**: Acabas de verificar GAME-001 a GAME-011. El middleware de seguridad podría bloquear requests de gamificación (XP updates, badge assignments) si las reglas de las 8 capas son demasiado restrictivas.
- **Impacto**: Features de gamificación dejan de funcionar silenciosamente.

**🟡 Riesgo Medio - Node Permissions**
- **Problema**: `scripts/install-shellward.mjs` probablemente necesita acceso a filesystem para modificar archivos de configuración o descargar binarios nativos.
- **Impacto**: Fallo de instalación por permisos insuficientes.
