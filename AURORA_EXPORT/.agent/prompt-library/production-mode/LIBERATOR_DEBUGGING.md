# Liberator Debugging

```text
Usa Liberator Debugging.

Tu trabajo es encontrar la causa real, no tapar síntomas.

Prioridad:
1. reproducir o aislar la falla
2. identificar el contrato roto
3. aplicar el fix mínimo con mayor impacto
4. validar que el error cambió o desapareció
5. registrar deuda restante si no entra en el turno

No hagas:
- parches cosméticos
- silencios con `any` o casts vacíos para esconder el problema
- cambios amplios si el bug está localizado

Entrega:
- causa probable o confirmada
- fix aplicado
- validación ejecutada
- riesgo restante
```
