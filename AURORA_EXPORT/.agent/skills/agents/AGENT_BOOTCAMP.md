# Agent Bootcamp

## Objetivo

Subir el nivel mínimo de cualquier agente nuevo para que trabaje como full stack profesional dentro de este proyecto.

## Duración

15 a 20 minutos de lectura y alineación antes de tocar código.

## Módulo 1: Entender el proyecto

Leer:

- `PROJECT_CHARTER.md`
- `NON_NEGOTIABLES.md`
- `ATTACK_PRIORITY.md`
- `INFRASTRUCTURE_PLAN.md`
- `STRUCTURE_CONTRACT.md`
- `VISUAL_STRUCTURE_CONTRACT.md`
- `FUNCTIONAL_STRUCTURE_CONTRACT.md`

Debes poder responder:

- cuál es el core del producto
- qué está roto o ambiguo hoy
- qué no se puede romper
- qué trabajo tiene prioridad real

## Módulo 2: Entender cómo se trabaja

Leer:

- `WORKSPACE_RULES.md`
- `AGENT_TASK_DIVISION.md`
- `AGENT_ROSTER.md`
- `AGENT_OPERATING_PROFILES.md`
- `IMMUTABLE_CORE.md`
- `WORK_COMPLETION_PROTOCOL.md`
- `DELIVERY_DISCIPLINE.md`
- `AGENT_ACADEMY.md`
- `AGENT_FUNCTION_MATRIX.md`
- `CLOUD_AGENT_FUNCTIONS.md`
- `../workspace/coordination/TASK_BOARD.md`
- `../workspace/coordination/CURRENT_FOCUS.md`

Debes poder hacer:

- reclamar tarea
- declarar foco
- respetar ownership
- dejar handoff limpio
- entender qué tipo de tarea conviene a tu perfil
- entender qué partes del sistema no se pueden tocar

## Módulo 3: Estándar técnico esperado

Un agente profesional aquí:

- no improvisa arquitectura
- no improvisa estructura visual o funcional
- no improvisa estructura operativa si ya existe una plantilla
- no mete hardcodes de entorno
- no toca archivos críticos sin reclamar
- no confunde cache con base de verdad
- no deja integraciones sensibles sin auth o validación
- no declara progreso ficticio

## Módulo 4: Estándar de ejecución

Antes de editar:

- identificar objetivo
- identificar impacto
- identificar archivos
- identificar riesgo de regresión
- elegir blueprint o template si crea algo nuevo

Mientras edita:

- mantener scope
- registrar avances
- no abrir trabajo paralelo oculto
- usar la plantilla correcta si debugea, integra, refactoriza, delega o cierra

Al cerrar:

- actualizar board
- actualizar focus si cambió
- dejar log
- dejar handoff si corresponde

## Examen rápido de salida

Antes de empezar a trabajar, el agente debería poder decir:

1. Qué tarea va a tomar.
2. Qué archivos va a tocar.
3. Qué no debe romper.
4. Cómo va a validar.
5. Qué deja documentado al terminar.

Si no puede responder eso, no está listo para editar.
