# Agent Function Matrix

## Objetivo

Traducir cada modelo a funciones concretas dentro del equipo.

| Agente | Mejor rol | Riesgo principal | Formato ideal de tarea |
|---|---|---|---|
| gpt-5 nano | supervisor barato | cerrar en falso o simplificar de más | coordinación, control, delegación |
| minimax M2.5 | auditor / refactor medio | abarcar demasiado | findings + ejecución acotada |
| minimax-m2.7 cloud | explorador técnico mejorado | ambigüedad alta | tarea con archivos y aceptación |
| bigpickle | implementador repetible | falta de criterio si el scope es vago | tarea corta y concreta |
| MIMO V2 PRO FREE | frontend guiado | inventar UI si no hay blueprint | sección o pantalla con spec |
| Gemini Flash | coordinador cheap | superficialidad | orden y síntesis |
| Gemini Pro | analista profundo | costo excesivo para tareas simples | alternativas y diseño |
| Claude Code | integrador / refactor complejo | sobreextensión del scope | integración o debugging largo |
| Gemma 3 4B | microtask worker | baja profundidad | transformaciones simples |
| Nemotron 3 | razonamiento estructurado | falsa seguridad | análisis técnico acotado |
| Ollama Qwen 3.5 | worker local | variabilidad | cambios repetibles |
| Kimi-K2.5 Cloud | contexto largo | sobreanálisis | síntesis amplia |
| Qwen 3.5 Cloud | implementador general | falta de control en tareas críticas | coding con spec |
| GLM5 Cloud | planner/documenter | difuminar límites | documentación y coordinación |
