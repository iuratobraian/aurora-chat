# Feature Blueprint

## Cuándo usarlo

Para cualquier feature nueva o refactor relevante.

## Blueprint

### 1. Problema

- qué problema resuelve
- por qué importa ahora

### 2. Usuario

- quién la usa
- quién la administra

### 3. Resultado

- qué cambia en la experiencia
- cómo se mide que funciona

### 4. Flujo

- entrada
- acción
- persistencia
- feedback
- salida

### 5. Datos

- source of truth
- cache local permitido
- eventos o side effects

### 6. Seguridad

- auth requerida
- validaciones
- riesgos de abuso

### 7. Operación

- logs
- errores visibles
- degradación si falla una integración

### 8. Validación

- smoke test
- edge cases
- criterio de done
