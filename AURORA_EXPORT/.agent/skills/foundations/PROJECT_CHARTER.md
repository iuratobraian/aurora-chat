# Project Charter

## Rol de este documento

Define la dirección del proyecto. Ningún agente debe inventar producto, prioridad o alcance fuera de este marco.

## Estado real del producto

La app tiene demasiado alcance simultáneo: comunidad, academia, señales, prop firms, marketplace, ads, pagos, PWA, automatizaciones e integraciones. Eso diluye valor, complica la UX y eleva el costo de mantenimiento.

## Objetivo de negocio inmediato

Convertir la app en un producto claro, confiable y vendible alrededor de un core:

- comunidad de trading
- herramientas útiles para trader
- academia y contenido premium
- monetización simple y creíble

## Objetivo operativo

Reducir ruido, aumentar claridad, mejorar conversión y ordenar la ejecución del equipo.

## Problemas prioritarios

1. Marca inconsistente.
2. Navegación sobredimensionada.
3. Home y comunidad sin jerarquía clara.
4. Pricing y monetización con poca claridad narrativa.
5. Documentación interna duplicada y contradictoria.
6. Shell de app y vistas clave demasiado cargadas.
7. Calidad y verificación insuficientes antes de declarar tareas como completadas.

## Decisiones de foco

- El core actual es `Comunidad + Herramientas + Academia + Planes`.
- Todo lo demás compite por atención y debe tratarse como secundario hasta que el core esté sólido.
- No se agregan features nuevas fuera del roadmap actual sin aprobación explícita.
- No se marca nada como completado sin evidencia funcional.

## Resultado esperado en 30 días

- marca unificada
- navegación simplificada
- landing/home coherente
- comunidad más usable
- pricing más claro
- backlog confiable
- ejecución multiagente sin duplicación
