# Mobile Game Factory System

## Objetivo

Crear una base profesional para construir juegos mobile con:

- estructura clara
- factory de producción
- IA integrada
- templates reutilizables
- prompts operativos
- monetización
- LiveOps

## Regla principal

No construir juegos como ideas sueltas.
Construirlos como productos operables.

## Capa mínima de un juego profesional

### 1. Core Loop

- acción de 5 a 15 segundos
- meta de sesión
- feedback claro
- fallo comprensible
- recompensa corta

### 2. Meta Loop

- progresión
- colección
- economía
- misiones
- upgrades

### 3. Retention Layer

- recompensas de vuelta
- eventos
- streaks
- contenido rotativo
- LiveOps

### 4. Monetization Layer

- IAP
- rewarded
- battle pass
- cosméticos
- bundles

### 5. Operability

- analytics
- remote config
- crash reporting
- content ops
- QA

## Estructura de creación

1. fantasy del juego
2. loop de 10 segundos
3. loop de 5 minutos
4. razón para volver mañana
5. sistema de progreso
6. economía
7. monetización
8. LiveOps
9. content pipeline
10. testing y soft launch

## Stack conceptual recomendado

- engine
- gameplay systems
- UI/HUD
- save/profile
- analytics
- remote config
- ads/IAP
- backend opcional

## IA integrada

La IA debe servir para:

- ideación de loops
- generación de specs
- tuning asistido
- eventos y LiveOps
- economy simulation
- QA y edge cases
- prompts de arte/video
