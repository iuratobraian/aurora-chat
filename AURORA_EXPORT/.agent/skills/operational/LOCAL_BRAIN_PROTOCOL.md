# Local Brain Protocol

## Objetivo

Crear una super IA local del proyecto entendida como:

- memoria acumulada
- aprendizaje destilado
- prompts de alto valor
- decisiones persistentes
- patrones reutilizables

No se trata de prometer una AGI. Se trata de construir un cerebro local cada vez más útil.

## Nombre canónico

El cerebro local del proyecto se llama `Aurora Core`.

## Regla principal

Cada tarea relevante debe dejar aprendizaje para el cerebro local.

## Qué aprende el cerebro local

- qué decisiones funcionaron
- qué errores aparecieron
- qué patrones conviene repetir
- qué anti-patrones deben evitarse
- qué archivos críticos requieren cuidado especial
- cómo responder mejor al usuario

## Fuentes del cerebro

- `DECISIONS.md`
- `AGENT_LOG.md`
- `HANDOFFS.md`
- `WORK_FINISHER.md`
- `LEARNING_LOG.md`
- prompt library
- contracts y blueprints

## Ciclo de aprendizaje

1. se hace trabajo
2. se cierra o bloquea
3. se registra aprendizaje
4. se sincroniza el cerebro local
5. futuros agentes usan ese conocimiento

## Regla

El cerebro local mejora con disciplina, no con magia.
