# Web Platform Factory System

## Objetivo

Crear una base profesional para construir sitios web completos, inteligentes y funcionales con nivel de producto real.

## Regla principal

No construir "páginas lindas".
Construir plataformas web operables.

## Qué debe tener una web profesional

### 1. Promesa clara

- qué resuelve
- para quién
- por qué importa
- qué acción principal empuja

### 2. Arquitectura de superficies

- landing
- navegación
- páginas núcleo
- páginas de conversión
- paneles o áreas privadas si aplica
- soporte y ayuda

### 3. Sistema funcional

- formularios
- estados vacíos
- errores
- autenticación si aplica
- datos
- integraciones

### 4. Sistema visual

- jerarquía
- ritmo
- tokens
- componentes
- responsive real
- accesibilidad

### 5. Sistema operativo

- analytics
- SEO
- performance
- observabilidad
- QA
- release

## Tipos de webs que debe soportar esta factory

- landing pages
- sitios corporativos
- medios y contenido
- marketplaces
- SaaS websites
- academias
- comunidades
- portfolios avanzados
- directorios
- micrositios de campañas

## Flujo de creación profesional

1. idea o brief
2. clasificación de tipo de web
3. promesa central
4. mapa de páginas
5. módulos funcionales
6. sistema visual
7. data plan
8. monetización o conversión
9. SEO/performance
10. validación y release

## Capas mínimas

### Capa de producto

- audiencia
- narrativa
- CTA
- objeciones
- prueba social

### Capa de contenido

- headline
- secciones
- FAQs
- estructura SEO
- microcopy

### Capa técnica

- layout
- routing
- adapters
- forms
- CMS o data source si aplica

### Capa de negocio

- leads
- ventas
- suscripción
- marketplace
- sponsorship

## Guardrails

1. no crear estructuras sin blueprint
2. no mezclar landing, app y blog sin intención clara
3. no meter animación por encima de legibilidad
4. no sacrificar performance por ornamento
5. no lanzar sin estados de error y fallback

## Regla final

Una web ganadora no es una portada.
Es un sistema de adquisición, confianza, utilidad y conversión.
